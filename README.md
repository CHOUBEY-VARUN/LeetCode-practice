# Java DSA Practice

A structured collection of Java solutions for Data Structures and Algorithms practice, focused on improving problem-solving ability for coding interviews.

This repository contains my accepted LeetCode submissions, organized by problem. Each folder includes the solution file and, where available, the generated problem statement/notes.

## Why this repository exists

I use this repository to track my DSA preparation and document my progress across common interview patterns such as arrays, strings, linked lists, trees, heaps, tries, sliding window, two pointers, binary search, recursion, and dynamic programming.

The goal is not just to collect solutions, but to build consistency, pattern recognition, and clean implementation habits in Java.

## Tech Stack

* **Language:** Java
* **Platform:** LeetCode
* **Sync Tool:** LeetHub v2

## Repository Structure

Each problem is stored in its own folder.

```text
problem-number-problem-name/
├── accepted-solution.java
└── problem-description.md
```

Example:

```text
347-top-k-frequent-elements/
├── 2026-06-16 07.11.20 - Accepted - runtime 16ms - memory 47.5MB.java
└── 347-top-k-frequent-elements.md
```

## How to Read the Solutions

Most solutions follow this structure:

1. Understand the brute-force idea.
2. Identify the reusable pattern.
3. Optimize for time and space complexity.
4. Implement the cleanest accepted Java solution.
5. Revisit important problems for retention.
